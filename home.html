Responsive Redesign: Write-Up

PART 1: IDENTIFYING USABILITY PROBLEMS

Picking a Website:

I picked a website for Hilltop Orchards & Furnace Brook Winery. This is a link to the website. Below is a screenshot of their homepage that I chose to redesign:

Finding Problems:

I identified the following problems with the webpage pictured above:

  • The homepage consists of a single large block of text. This text is in multiple colors and font sizes but there is not enough grouping. This makes learnability low.
  • The contrast between the text and the background in certain regions of the page is not high enough, making text hard to read.
  • The multiple hyperlinks embedded in the large block of text create issues with memorability. Because lots of unrelated information is grouped together without a label, users may not remember where to go for specific information.
  • Navigation bar text is rather small. It may be hard to read.

Accessibility:

WebAIM Wave detected the following accessibility concerns on the original webpage:
  • 4 empty links
  • 10 contrast errors (very low contrast)
  • 13 alerts
  • 14 structural element issues
  • 2 ARIA label issues
  • 3 feature issues

    • I could not get a screen reader to work so I used WAVE to simulate a screen reader. The screen reader did not recognise a large amount of important text on the page, such as the address and social media information, and COVID precautions. This poses an accessibility issue because it means that some users would lack critical information about the business.

    PART 3: RESPONSIVE REDESIGN

    Thankfully, changing the font size on my browser does not cause any of my webpage content to be compromised. I also used the Chrome Responsive Tool and my page remains responsive for the full range of sizes that my page must be responsive for. Using the WebAIM Wave tool, my page no longer presented the low contrast errors, feature issues, label issues etc that the original webpage presented.
